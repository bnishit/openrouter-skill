What You Can Do With This

Real problems. Concrete solutions.

01

A new model drops. You ship it same day.

A new image-capable model just appeared. You want to use it without rewriting the app or hardcoding another provider-specific path.
The skill fetches the live catalog, filters by capability, scaffolds the route, and ships today — not next sprint.

New models become same-day features, not backlog tickets.

02

Yesterday's OCR flow cost more than expected.

Your OCR pipeline processed 2,000 invoices. The bill is higher than expected. You need to look up exactly what one generation cost — not estimate from pricing tables.
The skill scaffolds a generation-lookup route that fetches a single generation by ID and returns its exact cost, provider, and token breakdown.

Post-hoc cost lookup per generation, not ongoing monitoring.

03

Find free and cheap models without scanning the API.

You want to find free or cheap models from the catalog and set up provider-aware fallbacks — without manually scanning the API.
The skill provides catalog-filtering helpers that surface zero-priced models and list providers, plus routing patterns for fallback configuration. It helps you discover and choose — not auto-route.

Catalog filtering and fallback patterns, not an auto-routing engine.

04

One integration for text, vision, images, and files.

You're tired of one SDK for chat, another for images, and a third for file workflows. You want one layer that handles everything.
The skill keeps everything on one OpenRouter-compatible layer — same auth, same routing, same cost tracking across all modalities.

Operational simplicity across every modality.

05

You need app icons and OG images. Now.

Launch is Thursday. You need icons, OG images, and social assets generated, previewed, and saved — without building a custom pipeline.
The skill ships presets for icons, OG images, and social assets. Generate, preview, and persist with full metadata.

Image generation as a product workflow, not a one-off demo.

06

Production rules without rediscovering them.

The feature works in dev. You don't want to remember all the small rules for fallbacks, tool loops, JSON validation, and cost logging.
The skill applies production playbooks for fallback ordering, tool-loop safety, JSON validation, and cost logging — encoded, not remembered.

Ship production-grade on the first deploy.

How It Works

Three steps. Then you are building.

1

Install

npx skills add bnishit/openrouter-skill

2

Invoke

Talk to your AI agent. Mention the skill and say what you need:

Use the OpenRouter skill to add image generation to this app. Use the OpenRouter skill to build an agentic tool-calling loop. Use the OpenRouter skill to add a model picker with free-model filtering. Use the OpenRouter skill to look up exact cost per generation. Use the OpenRouter skill to set up provider routing with fallbacks. Use the OpenRouter skill to add PDF and image chat support.
3

Ship

Your agent scaffolds production-ready code using the skill's knowledge:

  • Routes and API handlers from starter templates
  • Edge cases covered by production playbooks
  • Cost tracking and logging wired in
  • Fallback and routing configured
  • You review, test, deploy

Why This Exists

Most OpenRouter work is repetitive. This turns it into a reusable skill.

Teams usually redo the same OpenRouter integration steps every time: choosing a model, proxying requests, deciding on routing behavior, validating JSON, and wiring up multimodal inputs. OpenRouter Skill bundles those recurring patterns into one installable agent skill.

If someone searches for an OpenRouter skill, an OpenRouter agent skill, or an OpenRouter integration skill, this repository is designed to be the page that actually answers the question with working assets and concrete install commands.

FAQ

Is this only for one assistant?

No. The repository is packaged broadly for AI agents that support the skills install flow, and the landing page is intentionally written for general AI agent use.

What makes this an agent skill?

It packages reusable OpenRouter workflows, templates, references, and helper assets so an AI agent can apply them directly instead of rebuilding the same integration patterns from scratch.

What do I get besides docs?

You get the skill prompt, metadata, starter templates, smoke fixtures, shared helpers, and focused references that agents can use directly.
